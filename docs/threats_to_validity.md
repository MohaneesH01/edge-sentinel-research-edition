# Threats to Validity

This document collects the known limitations of the Edge Sentinel v1 evaluation
so reviewers can audit them. The paper's Section 7 will reuse this content.

## Internal validity

- **Synthetic-only data.** v1 evaluation is on synthetic fault injections
  generated by `backend/generate_dataset.py`. Synthetic data systematically
  under-represents sensor noise, environmental coupling, and partial-fault
  regimes. All reported accuracy / F1 numbers should be read as **upper
  bounds**.
- **Balanced classes.** 900 records per class. Real rooftop PV is
  heavily imbalanced: weeks of normal readings, hours of partial shading,
  rare sensor failures. The reported F1 will drop on real imbalanced data;
  expect 10-25% degradation in minority-class recall.
- **Single sensor model.** INA219 + DHT22 only. No MPU6050 vibration, no
  irradiance sensor, no panel-temperature sensor separate from ambient. This
  limits H1 (detection quality) and H3 (reproducibility).

## External validity

- **Single panel / single climate.** No evidence of cross-panel or
  cross-climate generalisation. H3 is not yet testable; this is the
  principal limitation of v1.
- **Single brand.** No panel-brand diversity. Different panels have
  different I-V curves, shunt behaviour, and thermal coefficients.
- **Static load assumption.** The synthetic data assumes a fixed load. Real
  installations have variable loads (appliances, inverters) that confound
  the current signature.

## Construct validity

- **Fault taxonomy.** Six classes (normal, open-circuit, partial shading,
  temperature anomaly, current deviation, sensor failure). Inherently
  approximate — e.g. partial shading and cell-crack are often
  indistinguishable from voltage/current alone.
- **Calibration not measured.** Edge Sentinel v1 does not measure whether
  the centroid model's softmax-like distance is a calibrated probability.
  H4 is partially addressed but not yet verified.

## Conclusion validity

- **Single train/test split.** Reported numbers are on a single 75/25 split.
  No k-fold cross-validation in v1.
- **No statistical significance test.** Differences between methods (rule
  vs centroid vs tree) are reported as point estimates; no confidence
  intervals. A 5x2 cross-validation with paired t-test is planned for v2.

## Reproducibility

- **Synthetic dataset generator is fully deterministic** (see
  `backend/generate_dataset.py`), but the seed is hard-coded. A CLI flag
  is planned.
- **No pinned dependency versions** beyond `requirements*.txt` floors.
  `pip freeze` artefact will be added in v2.
- **No Dockerfile.** Reviewers cannot reproduce the environment with one
  command. Adding this is on the v2 punch list.

## What mitigations are planned for v2

| Threat | Mitigation in v2 |
| --- | --- |
| Synthetic-only data | Bench captures from Steps 6-15 of `docs/Edge_Sentinel_Bench_Checklist.pdf`. |
| Class imbalance | SMOTE / class-weight rebalancing + stratified k-fold. |
| Single panel / climate | Multi-site capture protocol (`docs/cross_domain_protocol.md` planned). |
| No k-fold | 5x2 CV with paired t-test for method comparison. |
| No Dockerfile | One-line reproducibility (`docker pull ... ; docker run ...`). |
| Calibration | Temperature-scaled softmax in `ml/calibration.py`. |
